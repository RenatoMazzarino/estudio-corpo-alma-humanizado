# Forms (V2)

## Regras

1. Cada etapa principal usa card com cabecalho claro e corpo objetivo.
2. Campos editaveis usam superficie de input branca.
3. Blocos opcionais devem abrir por switch + accordion, sem poluicao inicial.
4. Validacao deve respeitar fluxo por etapa (cliente -> servico -> agenda -> financeiro -> observacoes).
